# Depth 1

## [ASSUMPTION]

- [ASSUMPTION] 현재 탐색 탭의 핵심 문제는 **상단 정보층 + 검색층 + 2단 필터층이 동시에 두껍게 쌓여**, 실제 제품 리스트가 너무 늦게 시작된다는 점이다.
- [ASSUMPTION] 이번 작업의 목적은 **탐색 탭을 설명형 화면에서 검색/비교 작업형 화면으로 되돌리는 것**이다.
- [ASSUMPTION] 대규모 재설계보다 **상단 구조 압축 + sticky 정책 재정의 + 2차 필터 조건부 노출**이 가장 큰 체감 개선을 만든다.

## [Core Reasoning]

이번 작업의 핵심은 하나다.

**항상 보여야 하는 것은 검색 바 + 1차 카테고리 탭만 남기고,
상단 정보 카드는 축약하며,
2차 영양소 필터는 조건부/온디맨드로 바꾸는 것.**

즉,

- 큰 정보 카드는 탐색 탭에서 비중을 줄여야 하고
- sticky는 최소한으로 써야 하며
- 필터 전체를 늘어놓지 말고, 필요한 순간에만 열리게 해야 한다.

## [Risks & Blind Spots]

- 상단 정보를 완전히 없애면 월령/시기 맥락이 약해질 수 있다.
- 영양소 필터를 너무 숨기면 건기식 탐색 접근성이 떨어질 수 있다.
- sticky를 여러 층으로 유지하면 다시 답답해진다.
- 홈 화면 문법을 탐색 탭에 그대로 남기면 제품 리스트 공간이 계속 죽는다.

## [Strategic Implications]

이 개편은 단순 미관 수정이 아니다.

실제 제품 데이터가 늘어날수록 탐색 탭은 더더욱 **작업 효율**이 중요해진다.

지금 상단을 정리해두면 이후:

- 제품 수 증가
- 필터 축 증가
- 비교 흐름 강화

가 들어와도 화면이 버틴다.

# Depth 2
## [ASSUMPTION]

- [ASSUMPTION] 탐색 탭의 본질은 **설명 화면**이 아니라 **빠르게 찾고 좁히고 비교하는 작업 화면**이다.
- [ASSUMPTION] 현재 답답함의 핵심은 “정보가 많아서”가 아니라 **상단 고정 정보층이 너무 두껍고, 검색/필터 층도 별도로 두꺼워서 결과 리스트가 너무 늦게 시작**된다는 점이다.
- [ASSUMPTION] 지금 단계에서는 대규모 재설계보다 **상단 구조를 압축하고, sticky 정책을 재정의하는 것**이 가장 큰 체감 개선을 만든다.

## [Core Reasoning]

### 1) 지금 구조의 문제는 명확하다

현재 탐색 탭 상단은 사실상 3층이다.

1. **정보 제공 Hero**
    - `26개월 · 유아 초기 시기`
    - 많이 살펴보는 영양소
    - 비교 탐색 설명
2. **검색 바**
3. **2단 필터**
    - 카테고리 탭
    - 영양소 탭

문제는 이 3층이 모두 위에 남아 있어서,

사용자가 탐색 탭에 들어왔을 때 실제 제품 리스트가 너무 늦게 나온다는 점이다.

탐색 탭에서 사용자가 원하는 건:

- 입력
- 필터
- 결과

인데, 지금은

- 설명
- 설명
- 필터
- 결과

순서다.

즉 현재 구조는 **홈의 문법이 탐색으로 침투한 상태**다.

홈은 맥락 설명이 중요하지만, 탐색은 **작업 밀도**가 중요하다.

---

### 2) “상단도 숨기고 하단도 숨기자”는 접근은 반쯤 맞고 반쯤 틀리다

스크롤 시 숨기고 나타나는 방식은 좋아 보이지만,

잘못 적용하면 오히려 더 불안정해진다.

왜냐하면 탐색 탭에서 진짜 중요한 건 “숨김”이 아니라 **우선순위 재배치**이기 때문이다.

지금 필요한 건:

- 무엇을 항상 보여줄지
- 무엇을 축약해서 유지할지
- 무엇을 필요할 때만 열게 할지

를 나누는 것이다.

즉 정답은

**“전부 숨기기”가 아니라 “상시 노출 1개 + 축약 유지 1개 + 온디맨드 1개”**다.

---

### 3) 내가 추천하는 구조: 3단 → 1.5단으로 줄여라

가장 좋은 방향은 이거다.

### 항상 보여야 하는 것

- **검색 바**
- **1차 카테고리 탭(전체 / 건기식 / 분유·우유·치즈)**

### 축약해서 유지할 것

- **월령/시기 정보**
    - 큰 Hero 카드가 아니라
    - 작은 컨텍스트 바 또는 pill 한 줄로 축약

### 숨겨야 하는 것

- **2차 영양소 필터 전체 행**
    - 항상 펼쳐두지 말고
    - 필요할 때 열리게 해야 한다

즉 이렇게 바뀌어야 한다.

### 현재

큰 정보 카드

- 검색 바
- 카테고리 탭
- 영양소 탭
- 결과

### 추천

작은 컨텍스트 바(축약)

- 검색 바
- 카테고리 탭
    
    (+ 필터 버튼 or 확장 시 영양소 칩)
    
- 결과

이렇게 해야 한다.

---

## [Risks & Blind Spots]

### 1. 정보 Hero를 완전히 없애면 탐색 맥락이 약해질 수 있다

그래서 완전 삭제는 아니다.

다만 탐색 탭에서는 큰 카드일 필요가 없다.

`26개월 · 유아 초기 시기 / 많이 보는 영양소: 칼슘·철·아연`

정도로 한 줄 요약만 남기면 충분하다.

---

### 2. 영양소 칩을 항상 숨기면 필터 접근성이 떨어질 수 있다

그래서 “완전 숨김”보다 **조건부 노출**이 낫다.

예:

- 카테고리 = 건기식일 때만 영양소 칩 노출
- 또는 `필터` 버튼 클릭 시 펼침
- 또는 스크롤 최상단에서만 보이고, 스크롤 후엔 접힘

---

### 3. sticky를 많이 쓰면 오히려 화면이 더 답답해진다

이건 중요하다.

보통 “고정이 편하겠지”라고 생각하지만, 모바일에서는 고정 영역이 두꺼워질수록 결과 공간이 죽는다.

탐색 탭에서 sticky는 **한 층만 무겁게**, 나머지는 가볍게 써야 한다.

## [Strategic Implications]

탐색 탭은 홈과 달라야 한다.

홈은 “이해시키는 화면”, 탐색은 “일하게 하는 화면”이다.

지금 탐색 탭이 답답한 이유는,

홈에서 잘 먹히는 월령 맥락 카드가 탐색에서도 같은 비중을 차지하기 때문이다.

이걸 고치면 단순히 예뻐지는 게 아니라:

- 제품 리스트 첫 노출이 빨라지고
- 검색/필터 사용성이 좋아지고
- 나중에 실제 제품 데이터가 늘어났을 때도 스캔 효율이 올라간다

즉 이건 UI 미세조정이 아니라

**탐색 탭의 역할을 “설명”에서 “검색/비교 작업”으로 되돌리는 작업**이다.

## [Immediate Actions]

내 추천안은 하나로 압축된다.

# 추천 구조

### A안. 가장 추천

### 1) 큰 상단 정보 카드는 없애고 “축약 컨텍스트 바”로 변경

상단 Hero를 아래 같은 얇은 바 1줄로 줄여라.

예:

- `26개월 · 유아 초기 시기`
- `많이 보는 영양소: 칼슘 · 철 · 아연`
- 우측 `i` 아이콘 또는 `자세히` 버튼

탭 초진입 시만 잠깐 펼쳐진 버전을 보여주고,

스크롤 시작하면 바로 이 축약형으로 접히게 하면 된다.

---

### 2) 검색 바 + 1차 카테고리 탭만 sticky 유지

항상 남길 건 딱 이 두 개다.

- 검색 바
- `전체 / 건기식 / 분유·우유·치즈`

이 두 개만 있으면 사용자는 계속 탐색 작업을 이어갈 수 있다.

---

### 3) 영양소 필터는 “항상 노출” 금지

지금 가장 비효율적인 건 `전체보기 / 칼슘 / 철 / 아연 ...` 줄이 계속 자리 먹는 것이다.

이건 아래 중 하나로 바꿔라.

### 추천 우선순위 1

- 카테고리가 `건기식`일 때만 노출

### 추천 우선순위 2

- `필터` 버튼 누르면 펼쳐지는 드롭다운/바텀시트

### 추천 우선순위 3

- 최상단에서만 보이고, 스크롤하면 접힘

내 추천은 **1 + 2 조합**이다.

즉:

- 기본은 숨김
- 건기식일 때만 필터 버튼 활성
- 누르면 영양소 칩 펼침

---

### 4) 적용된 필터만 “결과 위 active chips”로 남겨라

필터 UI 전체를 고정하지 말고,

현재 적용된 값만 결과 상단에 남기면 된다.

예:

- `건기식`
- `칼슘`
- `철`
- `26개월`

이렇게 active chip만 보여주면 현재 상태를 잃지 않으면서도 공간을 덜 먹는다.

---

### B안. 더 쉬운 타협안

개발 난이도를 낮추려면 이렇게 해도 된다.

- 큰 상단 정보 카드: 최상단에서만 보임, 스크롤 시 완전히 숨김
- 검색 바 + 1차 카테고리: sticky
- 2차 영양소 필터: sticky 아님, 상단에서만 보임

이 방식도 지금보다 훨씬 낫다.

다만 이상적인 건 A안이다.

---

## 내가 권하는 최종 판단

### 하지 말 것

- 상단 정보와 하단 검색/필터를 **둘 다 항상 고정**
- 스크롤 때마다 상단/하단이 둘 다 출몰하는 복잡한 애니메이션
- 필터 행 2개를 그대로 sticky 유지

이건 100% 다시 답답해진다.

### 할 것

- **정보층은 축약**
- **작업층은 검색 + 1차 탭만 상시 유지**
- **2차 필터는 조건부/온디맨드**
- **결과는 최대한 빨리 시작**

---

## 구현 우선순위

1. 상단 Hero → 축약 컨텍스트 바로 변경
2. 검색 바 + 1차 카테고리만 sticky
3. 2차 영양소 필터 숨김/조건부 전환
4. active filter chips 도입
5. 필요하면 나중에 부드러운 collapse animation 추가
